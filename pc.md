# 次に使うパソコンの調査

- MacMini
- メリット
  - 場所を取らない
  - コストの割にスペック高い
  - トータルコスト安い
- デメリット
  - 周辺機器が必要
    - ディスプレイ・キーボード・webカメラ・マイクなどついてない
    - touch系の処理はキーボードが必要
  - 外では作業できない

- 現在のMacについて
  - 下取りに出すとそれなりの値段で買いとってもらえそう
    - いくらで下取りできるかは要確認

- 考察
  - 周辺機器が揃っているので、MacMiniで良さそう
  - どの程度のスペックにするかは確認した方が良さそう
    - チップM2
    - メモリ16G
    - ストレージ256GB
    - 要確認

# メモリについて
- プログラミング用途の場合は16GB以上のメモリが推奨
  - 多くのプログラムやツールを同時実行してもスムーズに動作する
    - コードエディタやIDE
    - 仮想マシンやコンテナ
    - デバッガ
    - データベース
    - ブラウザ
      - 複数タブ
  - 大規模なプロジェクトや重い動作にも対応できる
  - 将来的なニーズにも対応できる

# チップについて
- M1チップ
  - コストを抑えつつ高性能なマシンを求めるならば、M1は優れた選択肢
  - 一般的なプログラミングや軽い作業には十分な性能を発揮
- M2チップ
  - 少し予算に余裕があり、将来的な拡張性やより重い作業を視野に入れている場合にはM2がおすすめ
  - M1よりもさらに高性能

# ストレージについて
- 256GB SSD
  - 軽いプログラミングや一般的な使用に最適。
- 512GB SSD
  - 中程度のプログラミングや複数のツールを使う場合にバランスが良い。
- 1TB SSD
  - 重いプログラミングやプロフェッショナル用途に最適。長期的な視点で安心。
- 2TB SSD以上
  - 非常に大規模なデータ処理やプロフェッショナル用途に最適。予算に余裕がある場合。
